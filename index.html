ORM

  • Object-Relational Mapping
  • Camada de Acesso a Dados
  • Tomar Conta de Boa Parte do Trabalho Pesado
  • Linguagem de Programação Orientada a Objetos
  • Bancos de Dados Trabalham com a Estrutura de Tabelas e Registros
  • ORM é a Camada que Transforma os Dados
  • Os Tipos de ORM Mais Comuns

Active Record

  • Objetos do Modelo
  • Desenvolvimento Mais Rápido e Leve
  • Grande Dependência Entre o Modelo e o Banco de Dados
  • Manutenção mais complicada:
    • Mudanças em uma das partes
    • Modelos Complexos, Mapeamento Complexo
    • Aumenta Duplicação de Código

Data Mapper

  • Camada de Objetos que Mapeiam
  • Código Mais Limpo e Simples de Compreender
  • Mapear uma Tabela do Banco de Dados
  • Os Objetos do Modelo Geralmente Estão Muito Interconectados
  • Pode Trazer Mais Dados do que o Desejado
  • Lazy Load:
    • Objeto Não Contém Todos os Dados
    • Sabe Como Obter os Dados

Estudo de Caso

Mapeamento

  • Annotations
  • Bloco de Comentário, o PHPDoc
  • Tabelas Mapeadas por Classes
  • Colunas mapeadas por Propriedades
<?php
namespace App\Models;
/**
 * @ORM/Entity
 * @ORM/Table(name=pessoa)
 */
class Pessoa {
    /**
     * @ORM/Id
     * @ORM/Generated
     * @ORM/Column(name=pessoa_id, type=int)
     */
    public $id;
    /**
     * @ORM/Column(name=nome, type=string, length=50)
     */
    public $nome;
    /**
     * @ORM/Column(name=data_nasc, type=date)
     */
    public $dataNasc;
}

Mapeamento

  • Analisar Annotations
  • Expressões Regulares
    • preg_match
    • preg_match_all
<?php
    $reflection = new ReflectionClass('App\Models\Pessoa');
    $doc = $reflection->getDocComment();
    $findAnnotatoins = '/@ORM\/[@A-Za-z0-9=,_\/\s\(\)\{\}]+/i';
    $findTable = '/Table\([@A-Za-z0-9=,_\/\s\(\)]+\)/i';
    $findName = '/name[\s]?=[\s]?(\w+)/i';

    preg_match_all($findAnnotatoins, $doc, $annotations);
    $annotations = join($annotations[0], '');
    preg_match($findTable, $annotations, $table);
    preg_match($findName, $table[0], $name);
?>
