ORM

  • Object-Relational Mapping
  • Camada de Acesso a Dados
  • Tomar Conta de Boa Parte do Trabalho Pesado
  • Linguagem de Programação Orientada a Objetos
  • Bancos de Dados Trabalham com a Estrutura de Tabelas e Registros
  • ORM é a Camada que Transforma os Dados
  • Os Tipos de ORM Mais Comuns

Active Record

  • Objetos do Modelo
  • Desenvolvimento Mais Rápido e Leve
  • Grande Dependência Entre o Modelo e o Banco de Dados
  • Manutenção mais complicada:
    • Mudanças em uma das partes
    • Modelos Complexos, Mapeamento Complexo
    • Aumenta Duplicação de Código

Data Mapper

  • Camada de Objetos que Mapeiam
  • Código Mais Limpo e Simples de Compreender
  • Mapear uma Tabela do Banco de Dados
  • Os Objetos do Modelo Geralmente Estão Muito Interconectados
  • Pode Trazer Mais Dados do que o Desejado
  • Lazy Load:
    • Objeto Não Contém Todos os Dados
    • Sabe Como Obter os Dados
